# Holging — Питч для инвесторов

## В одном предложении
**Inverse ETF для Solana.** Один токен, один клик, ноль ликвидаций.

---

## Проблема (рынок $47B)

Чтобы зашортить SOL сегодня, нужно:
- **CEX**: KYC, маржа, funding rate, риск ликвидации
- **Perp DEX** (Drift, Jupiter): сложно, высокий порог входа, позиция ≠ токен
- **Leveraged inverse tokens** (Ethereum): volatility decay, path dependency

**Не существует простого SPL-токена для обратной экспозиции к SOL.**

---

## Решение

**shortSOL** — SPL-токен, цена = k / SOL_price (мультипликативная модель 1/x)

- SOL растёт → shortSOL падает (и наоборот)
- **Нет volatility decay** — в отличие от leveraged ETF, без ежедневной ребалансировки
- **Нет path dependency** — цена зависит только от текущей цены SOL, не от истории
- **Нет риска ликвидации** — просто токен в кошельке
- **Нулевой slippage** — торговля по оракульной цене (Pyth Network)

---

## Стратегия Holging (уникальная IP)

50% SOL + 50% shortSOL = **математически гарантированная прибыль** при любом направлении цены

```
P&L = (x - 1)² / (2x) ≥ 0    for any x > 0
```

**8 теорем формально доказаны в Lean 4** — ни один DeFi-протокол на Solana не имеет machine-checked доказательств. Эквивалент «бессрочного straddle без theta decay».

## Вопросы и ответы

### Продукт

**В: Почему не просто продать SOL?**
Продажа SOL = выход из экосистемы. shortSOL = хедж внутри экосистемы. Holging (50% SOL + 50% shortSOL) позволяет зарабатывать на волатильности, оставаясь в экспозиции. shortSOL — это composable SPL-токен, который можно использовать в LP, фарминге, DeFi-стратегиях.

**В: Чем это лучше Drift/Jupiter perps?**
Разные аудитории. Perps = трейдеры (маржа, funding rate, мониторинг ликвидаций). Holging = холдеры (один клик, токен в кошельке, ноль обслуживания). Аналогия: ProShares Short S&P 500 (ETF) vs E-mini S&P futures.

**В: «Holging всегда в плюсе» — разве это не слишком хорошо, чтобы быть правдой?**
Математически доказано в Lean 4 (8 теорем). Экономически — точка безубыточности при движении SOL ±9% (0.40% roundtrip-комиссия). При дневных движениях 1–2% прибыль ≈ ноль. Стратегия зарабатывает на **волатильности** — чем больше движение, тем больше P&L. Это бессрочный straddle без theta decay.

**В: Что если SOL упадёт на 80%?**
shortSOL вырастет в 5 раз. Vault должен выплатить 5x. Circuit breaker приостанавливает работу при vault ratio < 95%. Решение: сверхколлатерализация. Формула: для защиты от -80%, vault = 5x TVL. Динамические комиссии автоматически вырастают до 0.80% при стрессе, замедляя оттоки.

### Экономика

**В: Кто является контрагентом?**
Vault обеспечен казначейскими средствами. Изначально сверхколлатерализирован командой/фондом. Комиссии (0.20–0.80%) накапливаются как буфер безопасности. При высоком объёме vault становится самодостаточным. V2: внешние LP с распределением доходности.

**В: Как финансируется пул ликвидности?**
Через инструкцию `add_liquidity` (только admin). Изначально: seed-капитал от фонда. V2: permissionless LP vault с распределением комиссий. V3: AMM-пул на Orca (SOL/shortSOL).

**В: Почему не блокировать mint при низком vault ratio?**
Mint **пополняет** vault (пользователь вносит USDC). Блокировка mint = блокировка притока ликвидности. Вместо этого динамические комиссии делают mint **дешевле** при низком ratio, стимулируя депозиты.

### Безопасность

**В: Как с аудитом?**
Devnet MVP. Аудит запланирован перед mainnet (OtterSec/Neodyme, бюджет $50K). Текущие защиты: checked arithmetic повсюду, 4-уровневая валидация оракула, circuit breaker, rate limiting, vault reconciliation, slippage protection, динамические комиссии, funding rate, two-step authority transfer, 16 кодов ошибок.

**В: Манипуляция оракулом?**
Pyth Network pull-based, 400ms. 4-уровневая валидация: staleness (120 сек), confidence (<2%), deviation (<15% mint/redeem, <50% update_price), floor ($1). Rate limiting (cooldown 2 сек) предотвращает sandwich-атаки. `update_price` не сбрасывает таймер rate limit.

**В: Один admin-ключ?**
Two-step `transfer_authority` + `accept_authority` уже реализован — новый ключ должен подписать для подтверждения. Интеграция Squads v4 multisig запланирована на Q1 2026 перед mainnet.

**В: Circuit breaker приостанавливает работу, когда пользователям больше всего нужно выкупить?**
Circuit breaker защищает от bank run — без него первый выкупающий забирает всё, остальным ничего не достаётся. Как страхование FDIC — ограничивает вывод для защиты всех. Решение: более глубокая сверхколлатерализация (vault ≥ 200–500% обязательств).

### Технология

**В: Почему Lean 4 для формальной верификации?**
Lean 4 — отраслевой стандарт для machine-checked доказательств (используется Microsoft, AWS). 8 теорем доказывают holging P&L ≥ 0, инвариант ценообразования, положительную гамму. Ни один DeFi-протокол на Solana не опубликовал формальные доказательства. Конкурентное преимущество для аудита и уверенности инвесторов.

**В: Масштабируется ли архитектура на мульти-ассет?**
Да. `pool_id` параметризован во всех 16 инструкциях. Для shortBTC: новый Pyth-фид + селектор пула на фронтенде. Архитектура готова, изменения в коде = 1 день.

### Рынок

**В: Squeeth (Ethereum inverse ETF) закрылся — почему у вас получится?**
Squeeth использовал модель x² (leveraged, path-dependent). Holging использует 1/x (без decay, без path dependency). Squeeth требовал сложную инфраструктуру и глубокую ликвидность. Holging обеспечен vault, нулевой slippage, более простая модель. Разные продукты для разных рынков.
